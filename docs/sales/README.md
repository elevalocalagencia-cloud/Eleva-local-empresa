## Fluxo recomendado

1. conduzir a primeira call com [DISCOVERY-SCRIPT.md](C:/Users/PEDROSO/Downloads/elevalocal-infra-sales/docs/sales/DISCOVERY-SCRIPT.md)
2. apresentar o deck base em [DECK-PILOT.md](C:/Users/PEDROSO/Downloads/elevalocal-infra-sales/docs/sales/DECK-PILOT.md)
3. enviar a proposta usando [PROPOSAL-TEMPLATE.md](C:/Users/PEDROSO/Downloads/elevalocal-infra-sales/docs/sales/PROPOSAL-TEMPLATE.md)
4. anexar o pacote legal indicado em `legal/`
5. alinhar CTAs e origem de leads com [LANDING-CTA-SPEC.md](C:/Users/PEDROSO/Downloads/elevalocal-infra-sales/docs/sales/LANDING-CTA-SPEC.md)

## Termos canonicos

- usar `automacao de atendimento via WhatsApp`; evitar `WhatsApp automation`
- usar `agendamento automatico` para produto e `agendamentos` para UI; evitar `agenda inteligente`
- usar `demo publica` em contexto comercial
- usar `portal do cliente` em vendas e `portal operacional` em contexto tecnico
- usar `piloto assistido` para oferta comercial; reservar `GO CONTROLADO` para gates internos
